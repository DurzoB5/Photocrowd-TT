The database contains three tables which consist of:

  • User
  • Competition
  • Submission
User

The User table holds data relating to a registered user of the system who may or may not of of submitted submissions to competitions.

Competition

The Competition table holds data about ongoing/previous competitions. As there were no readily available competition data in the JSON provided I used the second section of the submission name (after the `in`) to generate this data.

Submission

The Submission table is a through table which links a user to a competition with each submission being able to be given a name and a score.

Currently there is a restriction that a Submission name must be unique to the competition it is being submitted to.
